# 公益站自动签到 Chrome 扩展

公益站自动签到是一个 Chrome Manifest V3 扩展，用于管理多个公益 API 站点的每日签到。扩展支持NewAPI、Sub2API、ZenAPI等多类型站点，可自动识别站点类型、复用浏览器登录态，并在接口签到失败时打开填写的签到页链接查找签到按钮。

## 功能概览

- **多类型站点**：支持NewAPI、Sub2API、ZenAPI，以及可通过页面按钮完成签到的站点。
- **自动识别**：新增站点默认使用自动类型，签到时识别接口类型并记住结果。
- **linux.do OAuth**：NewAPI、ZenAPI 等站点在需要时可通过 linux.do OAuth 登录。
- **优先复用登录态**：签到前先检查浏览器已有登录态和缓存认证信息，只有不可用时才触发 OAuth。
- **页面兜底签到**：接口签到失败时，后台打开填写的签到页链接，查找“签到 / 签 / Check in”等按钮并点击。
- **访问模式**：支持只访问页面、不执行签到接口的站点。
- **余额展示**：每次签到或访问后尝试获取本轮最新余额；获取不到则不展示，也不会沿用旧余额。
- **静默后台执行**：自动打开的临时标签页保持后台，不抢焦点，结束后自动关闭。
- **站点名称获取**：站点名默认使用域名，运行时可从页面标题或元信息自动获取。
- **导入导出**：支持备份站点配置、签到页链接、站点类型、访问模式和每日签到时间。

## 安装

1. 下载或克隆本项目。
2. 打开 Chrome，进入 `chrome://extensions/`。
3. 开启右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择项目中的 `chrome-extension` 文件夹。

## 使用

### 添加站点签到页

1. 点击扩展图标。
2. 点击“添加站点签到页”。
3. 输入签到页链接，例如 `c.com/console/personal` 或 `https://c.com/console/personal`。
4. 如果该站点只需要每天访问页面，勾选“仅访问”。
5. 点击“添加”。

扩展会保存域名用于识别和接口请求，同时保存完整签到页链接用于列表跳转和页面兜底签到。

### 执行签到

- 点击“立即签到”可手动执行。
- 默认每天 09:00 自动执行，可在弹窗中修改“每日签到”时间。
- 没有站点或所有站点都被禁用时，“立即签到”按钮会置灰。

### 查看状态

站点列表会显示：

- **自动 / 访问**：自动接口签到或仅访问模式。
- **余额**：本轮签到或访问后成功获取到余额时显示。
- **待签 / 签到中 / 成功 / 已签 / 失败 / 失效**：最近一次结果。

余额只来自本轮任务。上一轮有余额但本轮没有获取到时，列表不会继续展示旧余额。

## 签到逻辑

1. 加载启用的站点。
2. 自动站点先识别类型：NewAPI、Sub2API、ZenAPI 等。
3. 获取站点名称：如果名称仍是域名，则尝试从页面标题或元信息更新。
4. 获取认证信息：
   - 优先使用缓存认证头。
   - 缓存不可用时检查浏览器已有登录态。
   - 仍不可用时，根据站点类型触发 linux.do OAuth 或读取本地 token。
5. 调用对应类型的签到接口。
6. 如果接口失败且适合页面兜底，后台打开签到页并查找签到按钮。
7. 按钮不可点击、页面显示已签到、接口返回已签到时，结果记为“已签”。
8. 签到结束后获取最新余额：
   - 优先从签到响应中解析。
   - 再尝试常见账户接口。
   - 如果走了页面兜底，则从兜底页面正文中识别。
   - 访问模式从访问页面正文中识别。
9. 保存本轮结果，关闭临时标签页。

## 支持的站点类型

- **NewAPI**：支持 `/api/user/checkin`、`/api/user/self` 等接口，支持 linux.do OAuth。
- **Sub2API**：支持 `/api/v1/user/check-in`，从本地存储读取 `auth_token`、`access_token`、`token` 等。
- **ZenAPI**：支持 `/api/u/checkin`、`/api/u/dashboard`，支持 linux.do OAuth。
- **页面签到站点**：接口失败后打开填写的签到页链接，自动查找按钮。
- **访问站点**：不执行接口签到，只定时访问页面。

## 导入导出

导出文件包含：

- 站点域名 `domain`
- 展示名称 `name`
- 是否启用 `enabled`
- 站点类型 `type`
- 模式 `mode`
- 签到页链接 `pageUrl`
- 每日签到时间 `autoSignTime`

导入旧版本配置时，如果站点缺少 `type`，会补为 `auto`，下次签到时重新识别类型并保存。

## 常见问题

### 为什么会触发 linux.do OAuth？

只有缓存和浏览器已有登录态都不可用时，才会按站点类型尝试 linux.do OAuth。已经登录且可复用的站点不会每次都 OAuth。

### 接口失败后会怎么处理？

扩展会尝试打开填写的签到页链接，在页面内查找签到按钮并点击。临时页面在后台打开，不抢焦点，结束后自动关闭。

### 页面失效会一直重试吗？

不会。页面是 Chrome 错误页，或接口返回 404/410 时，会标记为“失效”。

### 为什么没有余额？

余额是可选展示。只有本轮签到或访问能从响应、账户接口或页面文字中识别到余额时才显示。

## 目录结构

```text
chrome-extension/
├── manifest.json          # 扩展清单
├── background.js          # 后台签到、认证、页面兜底
├── popup.html             # 弹窗页面
├── popup.js               # 弹窗交互和列表渲染
├── config.js              # 站点配置读写和去重
├── schedule.js            # 每日定时逻辑
├── site-url.js            # 站点输入和页面 URL 处理
├── auth-headers.js        # 认证头工具
├── newapi-auth.js         # NewAPI 登录态判断
├── zenapi-auth.js         # ZenAPI OAuth 工具
├── checkin-result.js      # 签到响应解析
├── checkin-run-state.js   # 签到运行态
├── balance.js             # 余额解析
├── site-name.js           # 站点名称获取
├── page-status.js         # 页面失效判断
├── tab-options.js         # 临时标签页策略
├── render-guard.js        # 弹窗渲染并发保护
├── backup-config.js       # 导入导出
└── icons/                 # 扩展图标
```
