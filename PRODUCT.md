# Product

## Register

product

## Users

Users are people who manage multiple public API sites from a Chrome extension popup and want a quick, low-friction way to run or schedule daily check-ins.

## Product Purpose

The extension stores site check-in pages, detects compatible site types, reuses browser login state, performs manual or scheduled daily check-ins, and shows compact per-site status in the popup.

## Brand Personality

Calm, practical, compact. The interface should feel like a trustworthy utility that stays out of the way.

## Anti-references

Avoid marketing-page styling, decorative UI, unfamiliar controls for standard actions, and any changes that make the popup slower to scan.

## Design Principles

- Keep repeated site-management actions discoverable but compact.
- Prefer familiar product UI controls over novelty.
- Preserve existing workflows unless a requirement explicitly changes them.
- Make destructive or behavior-changing actions explicit with confirmation.

## Accessibility & Inclusion

Use native controls where possible, visible focus states, readable contrast, descriptive labels/titles, and reduced-motion-safe transitions.
