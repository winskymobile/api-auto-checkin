# 公益站自动签到助手 - Chrome 扩展

公益站自动签到助手用于在 Chrome 中管理多个公益 API 站点的每日签到。扩展支持NewAPI、Sub2API等多类型站点，并兼容 ZenAPI、页面按钮签到、仅访问页面等场景。

## 核心功能

- **自动识别站点类型**：新增站点默认自动识别 NewAPI、Sub2API、ZenAPI 等类型。
- **登录态优先复用**：优先使用缓存认证头和浏览器已有登录态，必要时才触发 linux.do OAuth。
- **接口签到**：根据站点类型调用对应签到接口。
- **页面兜底签到**：接口失败时后台打开填写的签到页链接，查找并点击“签到 / 签 / Check in”等按钮。
- **访问模式**：适合只需要每天访问页面的站点。
- **余额展示**：本轮签到或访问后获取到余额才展示；本轮没获取到就不展示旧余额。
- **后台静默处理**：临时标签页后台打开，不抢焦点，任务结束后自动关闭。
- **站点名称获取**：站点名默认用域名，运行时可从页面标题或元信息自动更新。
- **配置导入导出**：支持备份和恢复站点、类型、模式、签到页链接和每日签到时间。

## 安装

1. 打开 Chrome 扩展管理页：`chrome://extensions/`。
2. 开启“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本目录所在的 `chrome-extension` 文件夹。
5. 固定扩展图标，方便打开弹窗。

## 使用方法

### 添加站点签到页

1. 点击扩展图标。
2. 点击“添加站点签到页”。
3. 输入签到页链接，例如 `c.com/console/personal` 或 `https://c.com/console/personal`。
4. 如果站点只需要访问页面，勾选“仅访问”。
5. 点击“添加”。

扩展会保存域名用于自动识别和接口请求，同时保存完整签到页链接用于站点跳转和页面兜底签到。

### 手动签到

点击“立即签到”后，扩展会对所有启用站点依次执行签到。执行过程中会显示进度，站点状态会实时更新为“签到中 / 成功 / 已签 / 失败 / 失效”。

### 自动签到

默认每天 09:00 执行。可在弹窗底部“每日签到”时间选择器中修改并保存。

### 站点管理

- 点击站点名前的开关可启用或禁用。
- 点击站点名可后台打开签到页链接。
- 点击右侧删除按钮可删除站点。
- 没有站点或所有站点都被禁用时，“立即签到”按钮不可点击。

## 签到流程

1. 加载启用站点，并清除上一轮余额展示。
2. 自动站点先识别类型，识别结果会保存到配置中。
3. 如果站点名称仍是域名，尝试从页面标题或元信息获取显示名称。
4. 获取认证信息：
   - 使用缓存认证头。
   - 缓存不可用时检查浏览器已有登录态。
   - 仍不可用时，按站点类型触发 linux.do OAuth 或读取本地 token。
5. 执行对应类型的签到接口。
6. 接口失败时，打开填写的签到页链接进行页面兜底签到。
7. 页面显示已签到、按钮不可点击、接口返回已签到时，结果记为“已签”。
8. 签到结束后尝试获取最新余额。
9. 保存本轮结果，关闭临时标签页。

## 余额逻辑

余额只展示本轮获取到的值：

- 签到成功后会尝试获取余额。
- 今日已签也会尝试获取余额。
- 签到失败但仍有登录态或页面结果时，也会尝试获取余额。
- 页面兜底签到会在兜底页关闭前读取页面余额。
- 访问模式会从访问页面正文中识别余额。
- 如果本轮没有获取到余额，则不展示。

## 支持的站点类型

- **NewAPI**：常见接口为 `/api/user/checkin`，支持 linux.do OAuth。
- **Sub2API**：常见接口为 `/api/v1/user/check-in`，从本地存储读取 token。
- **ZenAPI**：常见接口为 `/api/u/checkin`，支持 linux.do OAuth。
- **页面签到站点**：通过填写的签到页链接查找按钮完成签到。
- **访问站点**：只访问页面并记录结果。

## 导入导出

导出内容包含：

- `domain`：站点域名。
- `name`：站点展示名称。
- `enabled`：是否启用。
- `type`：站点类型，通常为 `auto`、`newapi`、`sub2api`、`zenapi`。
- `mode`：站点模式，访问模式为 `visit`。
- `pageUrl`：签到页链接。
- `autoSignTime`：每日签到时间。

导入旧版配置时，缺少 `type` 的站点会补为 `auto`，下次签到时重新识别并保存。

## 权限说明

- `cookies`：读取目标站点和 linux.do 登录态。
- `storage`：保存站点配置、签到结果、认证缓存。
- `alarms`：每日自动签到。
- `tabs`：后台打开临时页面、站点页面和 OAuth 页面。
- `scripting`：在页面上下文中读取登录态、点击签到按钮、获取余额。
- `webRequest`：捕获站点页面发出的认证请求头。

所有数据保存在本地 Chrome Storage，不会上传到第三方服务器。

## 常见问题

### 为什么会弹出 linux.do OAuth？

只有缓存认证和浏览器已有登录态都不可用时才会触发。已登录且能复用的站点不会每次触发 OAuth。

### 为什么签到失败但没有留下页面？

自动处理使用临时后台标签页。失败后也会关闭页面，不再停留给用户手动操作。

### 为什么余额没有显示？

余额是可选信息。只有本轮能从响应、账户接口或页面文字中识别到余额时才显示。

### 页面失效会怎么处理？

Chrome 错误页、404、410 等会标记为“失效”，避免一直重复尝试。
